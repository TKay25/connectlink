  • Data Tier: PostgreSQL database with optimized schema for inventory and transactions

🤖 AI Classification System

How It Works

User enters product
→
AI analyzes name
→
Matches category
→
Returns confidence
→
Auto-fill or modal

Classification Methods (Priority Order)

Method Confidence Description Used For
Exact Match 95%+ Exact subcategory name match Known products with exact names
Keyword Match 50-80% Keywords found in product name "Cable" → Electrical category
Fuzzy Match 40-70% String similarity matching "LED Bulb" matches "LED Light"
Fallback 20% No match found, uses "Other" Completely unknown products

Confidence-Based Decision Logic

Confidence Category Exists User Action System Response
≥ 90% Yes None ✅ Auto-fill, green badge
50-89% Yes None ⚠️ Auto-fill, yellow badge
50-89% No Confirm 📋 Show modal, user decides
< 50% Any Optional ⛔ Red badge, manual entry suggested

🔌 API Documentation

1. AI Classification Endpoint

POST /api/ai/classify-product

Request: { "product_name": "LED Bulb 60W", "description": "Warm white, energy efficient" } Response: { "success": true, "classification": { "category": "Electrical", "subcategory": "LED Lights", "confidence": 92, "method": "exact_match" } }

2. Create New Category Endpoint

POST /api/categories/create

Request: { "category_name": "Smart Home Devices", "subcategory_name": "Smart Lights" } Response: { "success": true, "category_name": "Smart Home Devices", "subcategory_name": "Smart Lights", "message": "Category created successfully" }

3. Category Suggestions (Autocomplete)

GET /api/ai/category-suggestions?partial_name=led&limit=3

Response: { "success": true, "suggestions": [ { "name": "LED Lights", "category": "Electrical", "match_score": 95 } ] }

4. Test Classifier

GET /api/ai/test-classifier

Response: { "success": true, "test_results": [ { "product_name": "LED Bulb 60W", "classification": { "category": "Electrical", "subcategory": "LED Lights", "confidence": 92, "method": "exact_match" } } ] }

💾 Database Schema

product_categories Table (User-Created)

CREATE TABLE product_categories ( id SERIAL PRIMARY KEY, category_name VARCHAR(100) NOT NULL, subcategory_name VARCHAR(100) NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, UNIQUE(category_name, subcategory_name) );

connectlinkdatabase Table (Products)

Column Type Purpose
id INT Product identifier
name VARCHAR(200) Product name
category VARCHAR(100) Sub-category (AI classified)
buy_price DECIMAL(10, 2) Cost price
sell_price DECIMAL(10, 2) Retail price
stock INT Current stock level
unit_type VARCHAR(50) Unit of measurement
funding_source VARCHAR(50) "profit" or "capital"
